# 要件定義書

## 1. システム概要

「自己像 vs 他者像のギャップ可視化」をコアとした日本人向け性格診断Webアプリ。
ログイン不要で診断でき、友人に専用URLを送ることで他者からの評価も取得できる。

---

## 2. 機能要件

### 2-1. 自己診断機能
- 32問の5段階評価アンケートを表示する
- 質問は4軸（行動様式・対人距離・感情表現・価値基準）× 8問
- 回答完了後、16タイプのうち1つを判定する
- 判定結果を永続的に保存する
- 結果ページの専用URL（結果トークン）を発行する
- 友人診断用の別URL（友人トークン）を発行する

### 2-2. 友人診断機能
- 友人トークンURLを開くと「○○さんについて答えてください」と表示する
- 自己診断と同じ32問を3人称（○○さんは〜）で表示する
- 複数人が回答可能（人数上限なし）
- 有効期限なし
- 回答済みの友人数を結果ページに表示する

### 2-3. ギャップ比較・結果表示機能
- 自己回答と友人回答の平均を軸ごとに比較する
- 4軸それぞれのスコアを視覚的に表示する
- 自己タイプコード（例：FRKS）と他者タイプコードを表示する
- ギャップが大きい軸について解説文を表示する
- 風林火山（FRKS）タイプには特別な演出を表示する

### 2-4. シェア機能
- 結果URLをコピーするボタン
- LINEでシェアするボタン
- X（Twitter）でシェアするボタン
- 友人診断URLをコピー・LINEシェアするボタン

---

## 3. 非機能要件

### 3-1. パフォーマンス
- トップページの初回表示：3秒以内
- 診断結果の表示：2秒以内
- モバイルファーストで設計する

### 3-2. セキュリティ
- 結果URLは推測困難なUUIDベースのトークンを使用
- 友人診断URLは結果URLとは別トークンを使用（結果への直接アクセスを防ぐ）
- Supabase Row Level Security (RLS) を設定する

### 3-3. 可用性
- Vercel + Supabase の無料枠で運用開始
- 月間10万PVまでは無料枠で対応可能な設計

### 3-4. ユーザビリティ
- スマートフォン・PC両対応（レスポンシブ）
- 診断所要時間：5〜7分
- 回答UIはビジュアルな5段階スケール（左:いいえ 〜 右:はい）

---

## 4. 制約事項

- ログイン機能は初期リリースに含まない（拡張フェーズで対応）
- 結果URLを失うと再アクセス不可（初期リリース）
- メール登録は拡張フェーズで対応

---

## 5. 用語定義

| 用語 | 定義 |
|------|------|
| 結果トークン | 自己診断完了時に発行される結果ページ専用URL識別子 |
| 友人トークン | 他者診断用URLの識別子（結果トークンとは別） |
| 自己像 | 自己診断の回答から算出したタイプ |
| 他者像 | 友人診断の回答平均から算出したタイプ |
| ギャップ | 自己像と他者像の差分 |
